HTTP 重定向状态码详解 — 权重传递对照表

黑帽 SEO 核心技巧：选择不同的重定向状态码，决定搜索引擎是否将原页面的权重传递到目标 URL。 这是斗篷伪装中"爬虫走 301 到 SEO 页，用户走 302 到推广页"的关键。
状态码 名称 权重传递 请求方法 黑帽 SEO 用法
301 Moved Permanently ✅ 完全传递 GET → GET
POST → GET		 爬虫首选。将爬虫 301 到 SEO 优化页，原页面排名和权重完全转移。
示例：爬虫访问 /product/ → 301 → /seo-optimized-product/
302 Found ❌ 不传递 GET → GET
POST → GET		 用户跳转首选。将搜索引擎用户 302 到推广/变现页，原 URL 排名不受影响。
示例：用户从 Google 点击 → 302 → /landing-page/
303 See Other ❌ 不传递 POST → GET
（方法变更）		 SEO 中较少使用。主要用于 POST 后跳转 GET，避免表单重复提交。
不适合作为斗篷伪装的主要策略。
307 Temporary Redirect ❌ 不传递 方法不变
POST → POST		 API/表单场景。类似 302，但保证请求方法不变。用于需要保留请求方法的斗篷场景。
搜索引擎行为与 302 一致，不传递权重。
308 Permanent Redirect ✅ 完全传递 方法不变
POST → POST		 爬虫备选。类似 301，但保证请求方法不变。适用于需要保留请求方法的永久跳转。
搜索引擎行为与 301 一致，完全传递权重。
